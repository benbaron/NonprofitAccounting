1. Introduction

The Nonprofit Bookkeeping desktop application centralises the daily accounting operations performed by small to mid-sized nonprofit organisations. It combines ledger management, budgeting, donor tracking, inventory oversight, and reporting into a single JavaFX workspace backed by a plugin-friendly architecture.

This manual provides a guided tour of the application, outlines the standard operating procedures for each functional area, and shares best practices for safeguarding your financial data. Whenever code-level behaviour influences the user experience (for example, when certain menu items remain disabled until a company is open), those nuances are called out explicitly.

Application build: nonprofitbookkeeping 1.0-SNAPSHOT

Interface: JavaFX primary workspace with contextual Swing panels.

2. System Requirements

  • Operating System: Windows 10+, macOS 12+, or a modern Linux distribution capable of running JavaFX.
  • Java Runtime: OpenJDK/Oracle JDK 17 or later. The build uses Java 17 toolchains and JavaFX modules.
  • Memory: Minimum 4 GB RAM (8 GB recommended when working with large ledgers or JasperReports exports).
  • Disk Space: ~500 MB for the application, reports, and H2 database files, plus additional space for attachments or exports.
  • Permissions: Ability to read/write the company directory, including sub-folders for reports, backups, and document attachments.

3. Installation & Launch

  1. Install Java: Verify java -version reports JDK 17 or newer.
  2. Obtain the application: Clone the repository or download the packaged distribution zip provided by your deployment pipeline.
  3. Build (from source):
    • Open a terminal in the project root.
    • Run mvn clean package -DskipTests to build a shaded executable JAR.
    • The artifact will be written to target/nonprofitbookkeeping-1.0-SNAPSHOT.jar.
  4. Launch:
    • Execute java -jar target/nonprofitbookkeeping-1.0-SNAPSHOT.jar.
    • The primary window titled Nonprofit Bookkeeping appears, initially showing the Company Selection panel.

If you distribute pre-built binaries, provide the JAR together with the matching JavaFX runtime or bundle them using the platform-specific packaging tool of your choice.

4. Workspace Overview

4.1 Main Layout

The main stage uses a tabbed MainApplicationView. Tabs for Dashboard, Journal, Chart of Accounts, Reports, and Account Details are always present but disabled until a company file is open. When no company is selected the Company Selection panel occupies the centre area.

5. Getting Started Workflows

5.1 Create a New Company

  1. Choose Edit > Create or Edit Company.
  2. Follow the wizard to enter the organisation profile, fiscal settings, and base chart of accounts.
  3. When prompted, save the company file into a dedicated directory. The application enables the workspace tabs once the company is active.

5.2 Open an Existing Company

  1. Use Company > Open Company and browse to the company file (.json/.zip depending on your deployment).
  2. Confirm that the Dashboard tab activates and the window title reflects the open company.
  3. If you need to switch organisations later, choose Company > Close Company before opening a new file.

5.3 Record a Journal Entry

  1. Navigate to the Journal tab or use Edit > Edit Journal.
  2. Use the General Journal Entry panel to add lines with account, debit, credit, and memo information. Totals appear in the toolbar; imbalances trigger an alert when you attempt to save.
  3. Click Save to commit the transaction to the ledger. Entries immediately surface in the Account Details panel.

5.4 Maintain the Chart of Accounts

  1. Open the Chart of Accounts tab or select Edit > Edit Chart of Accounts.
  2. Use the tree to add, rename, or re-parent accounts. Invalid operations (duplicate codes, missing parents) trigger descriptive log entries and alerts.
  3. Save changes to persist them into the active company profile.

6. Module Reference

6.1 Dashboard

Summarises key balances, recent journal activity, and quick links. The skeleton implementation clears itself automatically whenever the company closes to avoid stale data.

6.2 Journal

Features the General Journal Entry panel and supporting tables for transaction review. Use filters to limit by date or account once implemented. The Undo Last Edit (SCA) action from the SCA Ledger plugin integrates with this tab.

6.3 Reports

Hosts JasperReports-driven outputs (Income Statement, Cash Flow, Trial Balance, and more). In the current build the tab displays a skeleton panel awaiting activation of the Jasper workflow, but menu commands and services are in place.

6.4 Account Details

Displays transaction history for a selected account, consolidating ledger entries generated from journal postings and imports.

6.5 Budget Editor

Accessible through Edit > Open Budget Editor, this JavaFX panel supports drafting and updating annual budgets. Launching requires an open company because it sources fund and account information from the active profile.

6.6 Operational Panels

  • Documents & Attachments: Stores supporting files linked to transactions.
  • Inventory & Depreciation: Tracks asset acquisition, depreciation schedules, and disposals.
  • Funds & Fund Accounting: Manages restricted/unrestricted fund balances.
  • Reconcile: Launches the bank reconciliation flow via LedgerReconcilePanelFX.

6.7 Development & Revenue Panels

  • Donors / Donations: Manage donor profiles and received gifts. Donations integrate with ledger posting rules to ensure audit trails.
  • Grants: Track awards, milestones, and expenditure requirements.
  • Sales & COG: Supports earned revenue tracking with cost-of-goods details.

6.8 Settings

The Settings panel surfaces application preferences, including default directories and optional integrations. Some fields are placeholders pending backend wiring—planned updates are tracked in TODO.md.

7. Data Management

7.1 Company Files

Company data is stored as structured JSON bundled with ledger attachments. Use Company > Save Company after significant changes, especially before closing the application.

7.2 Chart of Accounts Import/Export

Import a new or updated chart from Excel via Company > Import COA (XLSX). Export the current structure using Company > Export COA (XLSX) for backups or collaboration.

7.3 Statement Export

Select Company > Export Statement (OFX/QFX) to produce bank-compatible statements using the ledger. Validate the destination path before exporting; the application remembers the last used directory.

7.4 Database Operations

For advanced scenarios, use the Database menu to initialise or import an H2 database. Open/Create H2 DB… configures the storage location and ensures schema readiness. Import H2 script into DB… runs SQL scripts against the selected database.

7.5 Attachments & Documents

Within the Documents panel, upload supporting PDFs, receipts, or correspondence. Files are stored alongside the company directory; keep consistent backups to avoid orphaned references.

7.6 Backups

Use the Settings panel’s backup tab to capture point-in-time archives. Combine manual backups with source control or secure cloud storage for redundancy.

8. Plugins & Extensions

8.1 Plugin Discovery

At startup the application scans the plugin directory and lets each plugin contribute menu items through the shared menu bar. Plugin menus appear to the right of the core commands.

8.2 Sample Plugin

The Sample Plugin adds a Tools menu that demonstrates how plugins can inject UI actions by displaying a simple alert. Use it as a starting point for your own extensions.

8.3 SCA Ledger Plugin

The SCA Ledger plugin introduces a dedicated menu with utilities tailored to Spreadsheet Consolidation Accounting (SCA) workflows:

  • Load XLSM Table (SCA)
  • Import from Excel (SCA)
  • Import from JSON (SCA)
  • Save Modified Copy (SCA)
  • Input File (Generic – SCA context)
  • Output File (Generic – SCA context)
  • Undo Last Edit (SCA)

The plugin maintains its own working file context and integrates with the PageViewer panel to preview ledger pages after import.

9. Tips, Troubleshooting & Support

  • Menus stay disabled: Ensure a company is open. The application guards against actions that require company context.
  • Budget editor alerts: Open or create a company first; the panel validates its data sources before loading.
  • Plugin errors: Check the application logs. Plugin exceptions are caught and logged without crashing the core UI.
  • Report placeholders: Some report menu items are still under active development. Track progress in TODO.md or via the issue tracker.
  • Backups: Schedule regular backups using the Settings panel and off-site storage to protect against device failure.

For additional help, consult the in-app Help panel or escalate through your organisation’s support channel.
