This guide highlights the workflows that were recently restored to the application and provides developer notes for building consistent tooltips in the JavaFX UI.

Importing OFX/QFX bank statements

  1. Open the Account Activity view (Panels → Account Activity).
  2. Select the bank or credit card account whose transactions you need to import.
  3. Choose Import Statement (CSV/QIF/OFX) and pick the file.
  4. Confirm the format (OFX/QFX, QIF, or Excel) and map the transactions to the desired account.
  5. Imported entries are written to the company ledger and queued for reconciliation automatically.

Duplicate detection honours FITID values from OFX/QFX files and will only insert transactions that do not already exist in the ledger.

Exporting OFX/QFX statements

  1. Ensure a company is open and the ledger contains the transactions you wish to share.
  2. Navigate to Company → Export Statement (OFX/QFX).
  3. Select the account to export. Bank and credit-card style accounts are supported.
  4. Choose the target format (OFX or QFX) and the destination filename.
  5. The application writes an OFX 2.0 compliant file including balances, account metadata, and all dated transactions.

Charges in credit-card exports are written with negative amounts to match industry expectations, while payments appear as positive credits.

Importing legacy .npbk JSON data

Older releases distributed ledger extracts as JSON inside .npbk archives. To review those tables:

  1. Open the SCA Ledger menu that ships with the application.
  2. Select Import from JSON (SCA).
  3. Pick the JSON export. The viewer loads the rows into the embedded table so you can inspect or resave the data.

The importer keeps the original column order, respects mixed datatypes, and remembers the last directory for convenience.

Importing SCA Excel ledgers

To review the modern Excel version of the SCA ledger template:

  1. Open the SCA Ledger menu.
  2. Select Import from Excel (SCA).
  3. Choose the .xlsx workbook. The importer scans the sheet, expands allocation columns automatically, and streams the data into the viewer.
  4. The table opens in the Page Viewer window so you can sort, filter, or save a copy.

The importer remembers your last directory and keeps cells read-only so the grid reflects exactly what was stored in the workbook.

Developer appendix: JavaFX tooltip checklist

For a consistent and accessible experience, follow these practices when adding hover tooltips:

  • Centralise creation. Use a helper method to apply show/hide delays, wrapping, and max width.
  • Attach correctly. Call control.setTooltip for controls and Tooltip.install for generic nodes.
  • Style globally. Set common fonts, padding, and drop shadows in application.css.
  • Keep copy concise. Aim for one or two lines. Provide a secondary help view for lengthy explanations.
  • Support keyboard users. Show tooltips when the control receives focus, not only on hover.
  • Reuse instances. In table or list cell factories, create a single tooltip per cell and update its text to avoid churn.
  • Consider validation states. Pair pseudo-classes (e.g. .text-field:error) with context-sensitive tooltips for feedback.
  • Internationalise. Load tooltip strings from resource bundles and keep phrasing translation friendly.

The codebase includes helper snippets for dynamic tooltips, rich content (icon + text), and long-press behaviour on touch devices.
