Importing OFX/QFX Statements

  1. Choose File → Import Statement….
  2. Select your bank's .ofx, .qfx, .qif, or Excel ledger extract and press Open.
  3. Confirm the file format if prompted, then pick the destination account.
  4. Imported transactions are added to the ledger and queued for reconciliation. Duplicate detection runs automatically.

Tip: Importing into a liability account (for example, a credit card) automatically flips the sign of the amount so charges appear as withdrawals and payments as deposits.

Exporting OFX/QFX Statements

  1. Choose File → Export Statement….
  2. Select the account to export. Bank accounts and credit cards are supported.
  3. Pick OFX or QFX, choose a destination file, and click Save.

The export includes all ledger transactions for the chosen account with:

  • Accurate debit/credit direction based on the account type.
  • Transaction names sourced from “To/From” with memo fallbacks.
  • Auto-generated FITIDs, ledger balance, and statement date range.

Need Quicken compatibility? Choose QFX. The exporter adjusts sign-on metadata so Quicken recognizes the file immediately.

Opening Legacy .npbk (JSON) Files

Earlier versions of Nonprofit Bookkeeping saved company archives as plain JSON with a .npbk extension. The current loader supports both formats:

  • Modern archives — ZIP files containing modular JSON sections such as company_profile.json, ledger.json, chart_of_accounts.json, and a compatibility company_data.json.
  • Legacy archives — standalone JSON files (with or without the .npbk extension).

When you choose File → Open Company… the loader automatically detects which format you have and deserializes it accordingly. No conversion step is required.

JavaFX Tooltip Checklist

This checklist keeps hover tooltips clean, consistent, and accessible across the application.

1. Standardize with a Factory

public final class Tips {
  private Tips() {}

  public static Tooltip mk(String text) {
    Tooltip t = new Tooltip(text);
    t.setShowDelay(Duration.millis(500));
    t.setShowDuration(Duration.seconds(30));
    t.setHideDelay(Duration.millis(150));
    t.setWrapText(true);
    t.setMaxWidth(360);
    t.setAutoFix(true);
    return t;
  }
}

2. Attach Correctly

  • control.setTooltip(Tips.mk("…")); for UI controls.
  • Tooltip.install(node, Tips.mk("…")); for generic nodes.
  • Menu items aren’t nodes—consider mirroring the message in a status bar.

3. Global Styling

.tooltip {
  -fx-font-size: 12px;
  -fx-padding: 8 10;
  -fx-background-radius: 8;
  -fx-effect: dropshadow(gaussian, rgba(0,0,0,0.35), 10, 0.2, 0, 2);
  -fx-show-delay: 500ms;
  -fx-hide-delay: 150ms;
  -fx-show-duration: 30s;
}

4. Keep Them Short & Focused

  • Aim for two concise lines explaining the “why”.
  • For longer help, pair an info icon that opens the full Help window.

5. Rich Content Without HTML

Tooltip tip = Tips.mk("Drag files here to import");
tip.setGraphic(new ImageView("/icons/info.png"));
tip.setContentDisplay(ContentDisplay.LEFT);
button.setTooltip(tip);

6. Keyboard Accessibility

static void showOnFocus(Control c) {
  Tooltip tip = c.getTooltip();
  if (tip == null) return;
  c.focusedProperty().addListener((obs, was, is) -> {
    if (is) {
      Bounds b = c.localToScreen(c.getBoundsInLocal());
      tip.show(c, b.getMaxX() + 8, b.getMinY());
    } else {
      tip.hide();
    }
  });
}

7. Validation Tooltips

private static final PseudoClass PC_ERROR = PseudoClass.getPseudoClass("error");

static void bindErrorTip(TextField tf, ObservableStringValue errorText) {
  Tooltip errTip = Tips.mk("");
  errorText.addListener((o, oldV, v) -> {
    boolean has = v != null && !v.isBlank();
    tf.pseudoClassStateChanged(PC_ERROR, has);
    tf.setTooltip(has ? errTip : null);
    if (has) errTip.setText(v);
  });
}

8. Dynamic Tooltips

Tooltip tip = Tips.mk("");
tip.textProperty().bind(Bindings.format(
  "Runs %s items (shift-click = dry run)", selectedCountProperty()));
runButton.setTooltip(tip);

9. Table/List Cells

nameCol.setCellFactory(tc -> new TableCell<>() {
  private final Tooltip tip = Tips.mk("");
  private final Text measure = new Text();

  @Override protected void updateItem(String s, boolean empty) {
    super.updateItem(s, empty);
    setText(empty ? null : s);
    setTooltip(null);
    if (!empty && s != null) {
      measure.setFont(getFont());
      measure.setText(s);
      boolean truncated = measure.getLayoutBounds().getWidth() >
          (getWidth() - getPadding().getLeft() - getPadding().getRight());
      if (truncated) {
        tip.setText(s);
        setTooltip(tip);
      }
    }
  }
});

10. Disabled Controls

Button disabled = new Button("Pro feature");
disabled.setDisable(true);
StackPane wrapper = new StackPane(disabled);
wrapper.setPickOnBounds(true);
Tooltip.install(wrapper, Tips.mk("Upgrade to enable"));

11. Placement & Timing

  • Offset tips slightly so they do not obscure the control.
  • Use 300–700 ms show delay, 100–200 ms hide delay, 10–30 s duration.

12. Internationalization

  • Load strings from a ResourceBundle.
  • Keep sentences short to minimize reflow in translated UIs.

13. Performance Hygiene

  • Reuse tooltip instances; avoid creating new ones in mouse-move handlers.
  • Detach bindings in cell factories when cells become empty.

14. Testing

  • Use TestFX to hover nodes and assert tooltip text.
  • Capture snapshots at different scales and themes.
