# Reporting & Schedule Configuration Risk Register

| ID | Risk | Impact | Likelihood | Mitigation | Owner | Trigger |
|---|---|---|---|---|---|---|
| RSC-1 | Backfill creates non-unique `statement_section.section_code` after name normalization (space-to-underscore). | Migration fails or sections merge incorrectly. | Medium | Run precheck for duplicate normalized section names by `report_type`; introduce explicit curated `section_code` overrides before cutover. | Data Architect | Validation delta for sections is non-zero or unique constraint error occurs. |
| RSC-2 | Existing data includes `sign_policy` or `report_type` values outside constrained domain. | Forward migration aborts on check constraints. | Medium | Execute domain profiling query before deployment; stage corrective DML script in release runbook. | DBA | Constraint violation during insert into normalized tables. |
| RSC-3 | Application still writing legacy tables post-cutover causes drift from normalized tables. | Reporting outputs diverge; audit confusion. | High | Introduce temporary dual-write guardrail and compare parity daily until legacy writes are disabled. | App Engineering Lead | Validation parity delta appears after initial zero-delta baseline. |
| RSC-4 | Lack of trigger-based audit population leaves `config_change_event` empty after go-live. | Weak auditability and compliance gap. | Medium | Add app-service or DB triggers in phase 2; make audit insertion a release gate for configuration APIs. | Platform Team | No audit rows observed after approved config edits. |
| RSC-5 | Rollback drops normalized history tables needed for forensic analysis. | Loss of investigatory history after incident rollback. | Low | Export normalized tables and audit log snapshot before rollback; store in immutable release artifact bucket. | Release Manager | Rollback initiated in production window. |
| RSC-6 | Open-row uniqueness is validated by query (not hard-enforced with partial unique index in H2). | Duplicate active rules/mappings could be inserted by custom scripts. | Medium | Run validation SQL as a release gate; add DB-vendor-specific unique partial indexes where supported in production engines. | DBA + DevOps | Validation query #7 returns rows. |
